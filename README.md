# GitHub Repository Quality Analyzer

## Описание проекта

GitHub Repository Quality Analyzer — это инструмент для анализа качества GitHub-репозиториев с точки зрения оформления, воспроизводимости и готовности к использованию в портфолио.

Пользователь вводит ссылку на публичный GitHub-репозиторий, после чего приложение:

- клонирует репозиторий во временную папку;
- анализирует структуру проекта;
- проверяет наличие ключевых файлов и директорий;
- анализирует README;
- извлекает технологии, упомянутые в README;
- рассчитывает оценки по нескольким категориям;
- выдаёт рекомендации по улучшению проекта.

Проект ориентирован на студентов, junior-разработчиков и начинающих ML/Data Science специалистов, которые хотят улучшить свои pet-проекты перед добавлением в резюме.

## Основная идея

Хороший pet-проект — это не только код, но и понятная структура, документация, инструкция запуска и демонстрация результата.

Этот инструмент помогает быстро понять, насколько репозиторий готов к показу в портфолио.

## Что анализируется

### Структура проекта

Проверяется наличие:

- `README.md`;
- `requirements.txt` или `pyproject.toml`;
- `.gitignore`;
- `src/`;
- `tests/`;
- `notebooks/`;
- `app/`.

Также считается количество Python-файлов, Jupyter Notebook и общее количество файлов.

### README

Проверяется, содержит ли README:

- описание проекта;
- инструкцию установки;
- пример использования или запуска;
- описание технологий;
- результаты или метрики;
- изображения, графики или скриншоты.

### NLP-анализ README

Дополнительно используется NLP-based scoring для оценки качества README.

Подход состоит из двух частей:

- TF-IDF + cosine similarity между README проекта и эталонным описанием хорошего portfolio README;
- section coverage score, который проверяет наличие ключевых смысловых блоков: описание проекта, задача, данные, установка, запуск, технологии, результаты, структура проекта и дальнейшие улучшения.

Итоговый NLP README score помогает оценить, насколько README похож на полноценно оформленное описание проекта.

## Scoring system

Репозиторий получает оценки по категориям:

| Category | Description |
|---|---|
| Documentation | качество README и наличие важных смысловых блоков |
| Structure | наличие стандартной структуры проекта |
| Reproducibility | возможность воспроизвести проект и запустить код |
| Code | наличие исходного кода, структуры `src/` и тестов |
| Overall | итоговая оценка готовности репозитория |

Итоговый score рассчитывается как агрегированная оценка по категориям.

Для оценки документации используется гибридный подход:

- rule-based проверки важных секций README;
- NLP README score на основе TF-IDF similarity и coverage score.

Это позволяет учитывать не только наличие конкретных ключевых слов, но и общую близость README к хорошо оформленному portfolio README.

## Как запустить проект

### 1. Клонировать репозиторий

```bash
git clone https://github.com/denesis0-0/github-repo-quality-analyzer.git
cd github-repo-quality-analyzer
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить Streamlit-приложение

```bash
streamlit run app/streamlit_app.py
```

## Основные модули

- `src/repo_loader.py` — клонирование GitHub-репозитория во временную директорию.
- `src/file_scanner.py` — анализ структуры файлов и директорий.
- `src/readme_analyzer.py` — анализ содержания README.
- `src/readme_nlp.py` — NLP-based оценка качества README через TF-IDF similarity и section coverage.
- `src/tech_extractor.py` — извлечение технологий из README.
- `src/scoring.py` — расчёт оценок по категориям.
- `src/recommendations.py` — генерация рекомендаций.
- `src/report.py` — сбор общего отчёта.
- `app/streamlit_app.py` — пользовательский интерфейс.

## Ограничения

Текущая версия использует гибридный подход: rule-based проверки и простой NLP scoring на основе TF-IDF и coverage score.

Инструмент не оценивает качество кода на уровне AST, архитектуры или стиля программирования. Он анализирует в первую очередь оформление, структуру, воспроизводимость и готовность проекта к демонстрации.

NLP README score не является полноценной ML-моделью качества документации, но помогает дополнить rule-based анализ и сделать оценку README более гибкой.
